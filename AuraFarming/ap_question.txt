PURPOSE (MLO3)
The purpose of this assignment is to:
MLO3: Construct the program solutions using object-oriented programming language and approach.

Scenario
Developing a Smart AgriNepal Cooperative Management System
Background (Nepal Agricultural Context):
Agricultural cooperatives in Nepal play a crucial role in supporting farmers through crop planning, market
access, and advisory services. However, many cooperatives still rely on manual registers or basic
spreadsheets to manage farmer profiles, farm plots, crop plans, and daily market prices. This leads to
several operational challenges such as:
•
Loss of farmer records due to floods, fire incidents, or misplacement
•
Duplicate farmer registrations caused by inconsistent naming conventions
•
Crop planting in unsuitable Nepali seasons, resulting in reduced yields
•
Farmers selling produce without awareness of district-wise market prices
•
Lack of accountability due to absence of audit trails
•
Power cuts causing abrupt shutdowns and data corruption
To address these issues, the Government of Nepal (MoALD) has proposed an offline-first desktopbased solution that can operate reliably in rural areas with limited internet connectivity.

System Scenario
A cooperative named “Janasewa Krishi Sahakari” operates across Chitwan, Dang, and Rupandehi
districts.
•
Active Farmers: Approximately 1,200
•
Staff: 1 Admin (Agriculture Officer), 3 Cooperative Officers, multiple Farmers
•
Major Crops: Rice, Maize, Wheat, Mustard, Vegetables
Based on operational challenges observed in previous years, your software development team has been
hired to design and implement a Smart AgriNepal Cooperative Management System that works entirely
offline and stores all data locally using file-based persistence.

Objectives
•
Develop Programming Skills:
Gain practical experience in advanced object-oriented programming by implementing a real-world
cooperative management system using Java.

•
Enhance Data Handling and Persistence Skills:
Design and implement reliable file-based data storage using Java I/O streams for farmer records, crop
plans, market prices, and audit logs.

•
Improve Agricultural Decision Support:
Provide crop planning advisories based on Nepali seasons and basic rule-based analytics to reduce
crop-season mismatches.

•
Build a User-Centric Application:
Create an intuitive desktop-based graphical user interface using JavaFX or Swing to support
cooperative staff and farmers.

•
Practice Modular Development:
Divide the application into functional modules to promote code reusability and effective team
collaboration.

•
Apply Software Design Principles:
Utilize UML diagrams, object-oriented concepts, and design patterns to ensure maintainability and
clarity of the system.

You are required to assemble a team of Java developers (5-6 members) to design and implement a
desktop-based Smart AgriNepal Cooperative Management Application. The system must use an objectoriented approach to model the main application classes, their subclasses (if any), and their
corresponding attributes and methods. The application must include an appropriate graphical layout,
relevant data, and corresponding business logic.
Each member must contribute to the design of at least one class and/or subclass definition. In addition,
group members may collaborate with people from other faculties or industries who have the domain
knowledge to define the scope and coverage of your intended EV Management Application.

Important Constraint:
You may want to refer to the website here (Part 1-5) to kick-start your project.
•
Note that database usage is not allowed for this assignment. You must use Text or Binary File I/O or
Object Serialization for persistent data storage on a local machine. The Data Access Object (DAO)
pattern hence must be redesigned to cater for local file access.

•
You do not need to develop the web service endpoints. Instead, you should just develop the common
classes and methods in your application as a substitute for the endpoints. The API request will be
submitted by method calls.

•
For developing graphical user interface application, you may implement each of the above using
suitable UI components available in Java FX or Swing. Remember to document and justify your UI
selection.

The intended application should maximize the implementation of object-oriented concepts such as
instantiation, encapsulation, inheritance, abstraction, interface, polymorphism and design patterns such as
DAO, MVC(Model View Controller), DTO(Data Transfer Object) and others. The data storage method is
confined to what is covered in the syllabus, i.e., text, binary files and object files only and does not cover
various SQL, network, or cloud storage. You must strictly adhere to this rule. No marks will be awarded to
data storage methods that are not within the syllabus coverage.

2.1 Core modules
Module 1: User Management
Stores user accounts, authentication details, and role-based access control (Admin, Cooperative
Officer, Farmer).
Module 2: Farmer Profile Management
Manages farmer registration, profile updates, searching, and safe deletion with audit logging.
Module 3: Farm and Plot Management
Stores farm and plot details linked to farmers using a one-to-many relationship.
Module 4: Crop Planning and Advisory
Generates crop plans based on Nepali agricultural seasons and provides basic advisory outputs
using rule-based logic.
Module 5: Market Price Management
Maintains district-wise daily crop prices and supports historical trend analysis.
Module 6: Weather Risk Alerts
Loads weather data from files and generates alerts for potential agricultural risks.
Module 7: Reporting and Export
Generates reports such as farmer profiles, crop plans, and market summaries, exportable to files.
Module 8: Backup, Restore, and Audit Logging
Provides system backup and restore functionality along with append-only audit logs.
